# UI Reference Checklist

参考对象：`imgexe.com` 的「去除水印.exe」页面（以 `https://imgexe.com/zh/remove-watermark` 为准）  
当前对照对象：`图像工具箱/public/index.html` + `public/styles.css`

> 本清单只做 **QA / 参考对照**，不扩展功能；聚焦：**页面比例、按钮样式、控制面板密度、顶部栏 / 页脚设备感**。

## 已接近

### 1. 整体三栏工作台骨架已基本对上
- 参考站：左 rail / 中央工作台 / 右 rail 三栏结构。
- 当前实现：同样是 `120px + 主工作台 + 120px` 的三栏布局，视觉骨架和阅读顺序都接近。
- 实测接近点：
  - 参考站左右栏宽约 `112px`
  - 当前左右栏宽约 `120px`
- 结论：**页面主框架方向是对的**，不用重做结构，只需要继续压缩细节比例。

### 2. 中央工作台的“标题条 + 主编辑区 + 底部跑马灯”组织方式接近
- 参考站中央区是：
  1. 顶部黄色标题条
  2. 中间编辑/控制双栏
  3. 底部窄条信息带
- 当前实现也采用了相同分层：
  - `.panel-title`
  - `.editor-grid`
  - `.marquee-wrap`
- 结论：**大结构没跑偏**，可以在现有 DOM 上继续收紧尺寸与气质。

### 3. 编辑区左右比例已经接近参考站
- 参考站中央编辑区内部约为：
  - 画布区 `508px`
  - 控制区 `338px`
  - 比例约 `3:2`
- 当前实现约为：
  - 画布区 `491px`
  - 控制区 `304px`
- 结论：**控制面板在布局分栏上没有完全失真**，只是还不够“满”和不够“密”。

### 4. rail 按钮采用竖向图标 + `.exe` 文案，方向正确
- 参考站左右 rail 都是单个功能卡片式按钮，竖向排布，图标在上、文案在下。
- 当前实现也保持了同样模式。
- 结论：**信息架构接近**，主要差在按钮“工业设备感”而不是排版模式。

---

## 仍差距大

### 1. 页面纵向比例偏长，工作台显得“松”和“高”
- 参考站在 1200px 宽视口下：
  - 顶栏约 `70px`
  - 主区域约 `956px`
  - 页脚约 `45px`
  - 整页是比较紧凑、像桌面软件窗口的比例。
- 当前实现：
  - 顶栏约 `78px`
  - 主区域约 `1327px`
  - **没有页脚**
- 直接结果：
  - 当前页面比参考页**纵向多出一大截**，不够像“固定设备面板”，更像一张往下生长的网页。
  - 中央工作台高度被拉长后，控制区内容显得松散。

### 2. 顶部栏气质不对：现在更像“品牌 Hero”，参考站更像“设备头部”
- 参考站顶栏特征：
  - 高度更薄
  - 左侧 logo + 文案更克制
  - 右侧只有主题按钮和时间
  - 整体是很明显的“系统壳体 / 工具站机身”感觉
- 当前实现顶栏问题：
  - `brand-mark` 偏装饰化
  - 中间 `在线版 / 免安装 / 本地预览` 三个 pill 太像营销标签
  - 字重、留白、装饰元素都偏“展示页”，不是“工具设备页”
- 结论：**顶部栏设备感明显不足**。

### 3. 按钮样式太“UI 组件库”，不够像参考站的扁平系统按钮
- 参考站按钮特征：
  - 轮廓更硬、更平
  - 高亮按钮虽然用主色，但填充面积克制
  - hover / 阴影存在感弱，整体像软件按钮而不是 landing page CTA
- 当前实现按钮问题：
  - 全局 hover 有 `translateY(-1px)`，偏 Web 卡片交互感
  - `primary-btn`、`toolbar-btn.strong` 的大面积米黄色填充过重
  - 多处按钮边框都用了较强高亮描边，显得“发光感”偏强
  - rail 按钮的高度 `112px`，视觉块头比参考站更重
- 结论：**按钮风格还没完全进入 imgexe 那种“工业软件”语境**。

### 4. 控制面板密度不足，像“说明卡片集合”，不像系统侧边栏
- 参考站右侧控制面板观感：
  - 模块更紧
  - 标题行、分组间距更小
  - 垂直节奏连续，像密集参数栏
- 当前实现控制面板问题：
  - `.block + .block { margin-top: 18px; }` 间距偏大
  - 每个 block 的内部留白也偏多
  - `stats-block`、`tip-list`、`action-block` 都有较强卡片感
  - 最下面整块“快捷说明 / TIPS”很高，拉长了整栏节奏
- 结论：**现在是“内容齐全”，但不是“控制台密度”**。

### 5. 画布区前置了 hero-strip，削弱了参考站的工具属性
- 参考站中央区进入工作台后很快就到上传/编辑区。
- 当前实现在标题条下面放了一个较高的 `.hero-strip`：
  - 有英文 eyebrow
  - 有大标题 `上传 · 标记 · 修复 · 下载`
  - 有 3 个 badge
- 这块本身不难看，但它让页面更像专题页/落地页，不像 imgexe 的“直接开工”式工作台。
- 它也是当前页面纵向被拉长的重要原因之一。

### 6. 页脚缺失，导致整站完成度和“设备外壳闭合感”不够
- 参考站有明确页脚：
  - 隐私政策 / 用户协议 / 联系我们
  - `© 2026 图像处理工具箱 · POWERED BY DOINGFB`
- 当前实现完全没有 footer。
- 结果：页面底部是以跑马灯直接结束，像 demo，而不是完整产品页。

## 本轮必须处理

### 1. 先把纵向比例压回去，优先恢复“固定工作台”观感
**必须做的原因：** 这是和参考站差异最大的第一视觉来源。  
建议优先动作：
- 压缩 `.hero-strip` 高度，甚至考虑弱化为更薄的信息条
- 收紧 `.frame-sub` 的外边距（当前 `18px` 偏松）
- 收紧控制面板各 `.block` 间距与内边距
- 控制中央工作台总高度，不要继续向下拉成长页

### 2. 顶栏去营销化，改成更薄、更硬的“设备头部”
**必须做的原因：** 顶栏是第二个最影响相似度的区域。  
建议优先动作：
- 弱化或移除中间三枚 `mini-pill`
- 收窄顶栏高度与内边距
- 减少 `brand-mark` 的装饰存在感
- 让左侧品牌、右侧时间更像工具壳体上的状态信息，而不是网站宣传头图

### 3. 按钮统一收敛：减少大面积高亮、减少浮动感、减少“卡片 CTA”气质
**必须做的原因：** 当前很多局部已经像了，但按钮一眼就暴露“不是参考站”。  
建议优先动作：
- 去掉或减弱全局 `button:hover` 的 `translateY(-1px)`
- 降低主按钮实色填充面积，让激活态更接近“系统按钮高亮”而不是大 CTA
- rail 按钮、工具条按钮、分段按钮统一成更硬朗的边框与更紧的高度
- 减少阴影和发光感，提升“扁平工业 UI”一致性

### 4. 控制面板必须做“密度回收”
**必须做的原因：** 右侧栏当前信息完整，但节奏太松，直接影响参考感。  
建议优先动作：
- 缩小 section label、按钮、range、tip 之间的垂直间距
- 把 `快捷说明 / TIPS` 缩短或折叠，不要占据整块高度
- 让 action 区与上方设置区连接得更紧，形成连续控制流
- 目标不是加内容，而是让右栏更像 `控制面板.SYS`

### 5. 补一个参考站风格页脚，封住页面底部
**必须做的原因：** 这是最低成本、最高收益的“完成度修正”。  
建议优先动作：
- 增加一条薄 footer
- 文案结构参考原站：链接组 + 版权行
- 视觉上保持低调，不要抢工作台主体

---

## 一句话判断

当前版本已经有 **“imgexe 工作台 clone” 的骨架**，但还没完全进入参考站那种 **紧凑、克制、像桌面工具壳体** 的状态。  
本轮最该动的不是功能，而是：**压缩纵向比例、收紧按钮系统、提高右栏密度、补足顶部/页脚的设备感闭环。**
