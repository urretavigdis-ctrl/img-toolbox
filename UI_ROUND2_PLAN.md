# UI Round 2 Plan — 去除水印.exe

目标：只做 `imgexe.com`「去除水印.exe」页面的高相似 UI 精修；不扩展新功能，不改交互链路，只收敛视觉与版式。

## 1) 页面比例
- 把主工作区从当前偏“铺满屏幕”的响应式大板，收紧为参考站那种**居中、偏窄、纵向更高**的设备感画布。
- 调整 `.layout` / `.workbench` / `.editor-grid`：减少中间主区最大宽度，左右 rail 与中区之间留出更明确留白，避免现在横向过松。
- 提高主面板整体高度占比，压缩 hero 区、标题区、顶部辅助信息的高度，让「画布 + 右侧控制板」更早进入首屏主体。
- 让左画布区与右控制区比例更接近参考页：**左侧约 62~66%，右侧约 34~38%**，不要再让左区显得过宽、右区过胖。

## 2) 按钮与控件样式
- 全面收敛按钮视觉：从当前偏“现代玻璃拟态”改为参考站的**硬边、薄描边、浅金实底 + 黑字 / 黑底 + 浅描边**体系。
- 主按钮、分段按钮、格式按钮统一改成更扁平的矩形，减弱 hover 位移、阴影、圆角、发光效果。
- `seg active`、`primary-btn`、`toolbar-btn.strong` 统一成参考站的浅金高亮块；未激活按钮保持黑底细边，文字更亮、更紧凑。
- range、状态卡、meta chip 也要同步去“卡片感”，减少背景层次和内阴影，改成更像工业控制面板的线性控件。

## 3) 空态上传区与控制面板密度
- 空态上传区改成参考页的**纯黑大画布 + 中心弱提示**：去掉现在过强的 badge、说明文案、装饰纹理，保留极简图标、主提示、副提示。
- 上传区底部加回更接近参考图的**底边操作条/底部重选入口**，避免当前上传按钮全部堆在顶部工具条里导致空态重心不对。
- 右侧控制面板做“高密度压缩”：缩小 section 间距、按钮高度、说明文字行距，让 Brush / Mode / Format / Action / Tips 更像一块连续的系统面板。
- 状态信息从三张并列 stat-card 收敛成更轻、更窄的信息区，减少当前信息块的“卡片化”和视觉分裂。

## 4) 顶部栏与页脚设备感
- 顶栏改成参考站的**细框横条 + 中央站点名 + 右侧时间模块**，弱化现在的副标题、彩色 badge、品牌装饰块。
- `brand-mark` 与顶部中部内容改得更“设备铭牌化”：强调线段、对称、字距和机械感，而不是当前偏品牌展示的 logo 组合。
- 底部补足参考页那种**细线分隔 + 低位页脚信息 + 工具站声明文字**，并把跑马灯的存在感降下来，避免喧宾夺主。
- 整体统一“黑底、浅金、细描边、低饱和”基调，减少 ambient 光斑、背景辉光、blur 玻璃感，让页面更像参考站的复古终端设备。

## 实施顺序
1. 先改全局骨架比例：`topbar`、`layout`、`workbench`、`editor-grid`、`dropzone` 高宽关系。
2. 再统一按钮/分段控件样式 token：边框、底色、字号、hover、active。
3. 然后压缩右侧控制面板与空态上传区文案/层级。
4. 最后处理顶部栏、底部栏、背景氛围，收尾整体设备感。
